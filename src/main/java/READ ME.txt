Данное решение в ReportService относительно удачное. Оно позволяет быстро внести новые варианты отчета,
без нарушения логики програмы. Для этого необходимо создать новый класс и зарегестрировать в fillInstructions(map)
Если варианты отчетов перевалят за 1000, будет создано 1000 классов внесенных в fillInstructions(map).
